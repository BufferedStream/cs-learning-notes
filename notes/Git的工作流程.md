## Git的工作流程

Git 作为一个源码管理系统，不可避免涉及到多人协作。

写作必须有一个规范的工作流程，让大家有效地合作，使得项目井井有条地发展下去。“工作流程”在英语里，叫做“workflow”或“flow”，比喻项目像水流那样，顺畅、自然地向前流动，不会发生冲击、对撞、甚至漩涡。

![img](http://www.ruanyifeng.com/blogimg/asset/2015/bg2015122301.png)

本文介绍三种广泛使用地工作流程：

- Git flow
- Github flow
- Gitlab flow
